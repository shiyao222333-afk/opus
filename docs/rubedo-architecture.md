## 一、共用工具层（任何 SOP 都能用）

| 工具名 | 功能 | 服务 SOP | 状态 |
|--------|------|---------|------|
| **时间记录工具** | 做完一单点几下，自动计算耗时 | 所有计时类 SOP | 📋 规划中 |
| **AI 对话工具箱** | 需求确认清单、报价计算、合同生成 | 所有接单类 SOP | 📋 规划中 |
| **数据分析工具** | 自动汇总时间/利润/流量数据，生成周报 | 所有数据类 SOP | 📋 规划中 |
| **自动化任务调度** | 定时提醒记录、定时跑数据分析 | 所有需要提醒的 SOP | 📋 规划中 |

**特点**：不绑定具体业务，任何 SOP 都能用。

---

## 二、SOP 独特工具层（每个副业项目自己的专属工具包）

### 酷家乐 SOP 工具包

| 工具名 | 功能 | MVP 阶段 | 状态 |
|--------|------|---------|------|
| **RPA 抢单脚本** | 监控淘宝接单群，新单出现→系统通知 | 第5-6周（月20单验证） | 📋 规划中 |
| **2D 布局模板生成器** | 根据户型自动生成2D布局草图 | 二期 | 💭 远期 |
| **效果图渲染批量处理器** | 批量渲染多个视角 | 二期 | 💭 远期 |
| **AI 需求确认清单生成器** | 客户来了→填选项→生成标准化需求文档 | 第1周就能做（手动版） | 📋 规划中 |

### 小红书 SOP 工具包

| 工具名 | 功能 | MVP 阶段 | 状态 |
|--------|------|---------|------|
| **AI 选品工具** | 分析爆款笔记，推荐适合代发的商品 | 第1周 | 📋 规划中 |
| **AI 笔记生成器** | 标题+正文+标签，批量生成 | 第1周 | 📋 规划中 |
| **RPA 自动发布工具** | 自动登录+自动发布笔记 | 第二刀通过之后 | 📋 规划中 |
| **流量数据分析工具** | 自动抓浏览/点赞/评论数据，分析爆款规律 | 第2周之后 | 📋 规划中 |

### （未来其他 SOP 工具包）

| SOP | 可能包含的工具 |
|-----|----------------|
| **未来 SOP** | 每个新 SOP 一套工具包，不混在一起 |

## 三、内容发布（副产品）

| 平台 | 说明 |
|------|------|
| B站 | 过程即内容（酷家乐画图过程→B站视频） |
| 小红书 | 酷家乐效果图展示 + 小红书 MVP 笔记 |
| 微信公众号 | 深度文章（AI 生成） |
| 个人网站 | 作品集 + 博客 |

**注意**：内容发布是副产品，不是 Rubedo 的核心。核心是**让副业 SOP 自动运转**。

## 四、工具开发优先级（基于 MVP 时间线）

| 优先级 | 工具 | 什么时候需要 | 说明 |
|--------|------|------------|------|
| 🔴 P0 | **时间记录工具**（最小化版） | 酷家乐 MVP 第1周 | 第1周就要做时间审计，有个工具比手动填表快很多 |
| 🔴 P0 | **AI 需求确认清单生成器**（手动版） | 酷家乐 MVP 第1周 | 客户来了就能用，不需要写代码 |
| 🟡 P1 | RPA 抢单脚本 | 酷家乐 MVP 第5-6周 | 需要月20单，RPA 提醒能提高效率 |
| 🟡 P1 | AI 选品工具 + AI 笔记生成器 | 小红书 MVP 第1周 | 和小红书 MVP 同步开始 |
| 🟢 P2 | RPA 自动发布工具 | 小红书 MVP 第二刀通过后 | 第一刀手动发，验证流量后引入 |
| 🟢 P2 | 数据分析工具 + 自动化任务调度 | MVP 全部通过后 | 规模化阶段才需要 |

## 五、与 OpusMagnum 的关系

```
OpusMagnum（总指挥部）
        │
        ├── Citrinitas（知识引擎）—— 存储知识
        ├── Nigredo（内容提炼）— 视频→文档
        ├── Albedo（矛盾检测）— 验证可信度
        └── Rubedo（自动化平台）—— 执行 SOP
                │
                ├── 共用工具层（任何 SOP 都能用）
                └── SOP 独特工具层（每个副业一套工具包）
```

**Rbedo 的地位**：不是"内容发布工具"，而是**让副业 SOP 自动运转的平台**。
